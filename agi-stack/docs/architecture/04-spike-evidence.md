# 04 · 决策 Spike 已验证结论

> 本架构不是空想:其两条主轴([01](01-portable-core.md) 可移植核心、[02](02-extensibility.md) 可扩展性)均已用**可运行、可测试**的 Rust Spike 证伪通过;第三主轴的**热插拔**能力([06](06-agent-core-design.md)/[07](07-plugin-runtime-architecture.md))亦已由跨层热插拔 demo 证伪(见 #9);**控制流/数据流分离**([08](08-control-data-plane-separation.md))由 CP→DP xDS 风格 reconcile demo 证伪(见 #10)。Spike 代码在仓库根 `spikes/rust-portable-core/`(`README.md` 含完整命令与结论)。**Phase 1 已把这些模式提升为生产级 Cargo workspace**(本根 `agi-stack/` 下的 `crates/` + `apps/`,`cargo test --workspace` 25 测试绿、服务器全端点 curl 验证,含会话崩溃恢复——见 #11);移动端**两端均已产出真实设备产物**:Android `.so`(#12)、iOS XCFramework 并在 iPhone 17 模拟器实跑(#13)。

切片选定:**Episode → Memory 提取 + 语义存储/检索**,锚定真实代码 `src/domain/model/memory/{episode,memory}.py`、`src/domain/ports/repositories/memory_repository.py`。故意覆盖所有跨平台难点:纯领域实体、存储端口双实现、一步 async LLM 边界、语义检索、跨 WASM/UniFFI 序列化。

## 4. 总评

make-or-break 风险(**运行时无关核心 → 一份代码,server + 浏览器 + 设备**)已**确认**,有跨 server / 浏览器-WASM / 嵌入式 DB / 原生移动绑定的可运行、可测试产物,**移动两端(Android `.so` + iOS XCFramework)均已交叉编译为真实设备产物,iOS 更在 iPhone 17 模拟器内实跑通过**(见 #12/#13)。**PC 桌面外壳**(Tauri)亦跑通:同一核心 + SQLite 设备适配器链入桌面外壳、`ingest`/`search` 命令经 `invoke` 调用、命令逻辑无头可测(见 #17)。**可扩展性轴**亦确认:不可信工具经 `ToolHost` 端口沙箱化,其 WASM 宿主编到每个目标(含浏览器核心)。**热插拔轴**进一步确认:工具表 `ArcSwap` 原子换 + 扩展清单 enable/disable + 形态分类均跑通,且飞行中轮次按轮次边界见旧版本(确定性强于 OpenClaw 的重启式模型,见 [07](07-plugin-runtime-architecture.md))。**控制流/数据流分离轴**亦确认:控制面 SSOT + 数据面声明式 level-triggered reconcile + xDS 风格 version/nonce + ACK/NACK + last-good 跑通,reconciler 纯同步(无 tokio)故端上同实现;此即 Istio 的 Rust 数据面 ztunnel 同构机制(见 [08](08-control-data-plane-separation.md))。**不可信工具运行时**进一步生产化:server/桌面换入 **Wasmtime** 宿主,以 **fuel + epoch 双配额**硬约束不可信 wasm guest,契约升格为 typed `wit/tool.wit`,iOS/浏览器留 Wasmi 兜底(见 #18)。**端上向量检索**亦退役玩具实现:设备索引换入纯 Rust **HNSW 近似最近邻**(`instant-distance`,非 `sqlite-vec` C 扩展以保跨端可移植),embedding 升为高维字符 n-gram 哈希 —— N=10k 实测 P50 **2.43 ms**(暴力 75 ms,31× 加速),稳达 ≤20 ms 阈值(见 #19)。**AI/LLM 抽象轴**亦从 stub 落到真实端点:`LlmPort`/`EmbeddingPort` 新增打 OpenAI/LiteLLM 兼容端点的 HTTP 适配器,cloud↔local 经 DI 环境变量切换(缺省离线 stub、零 key),mock-server 测试全绿,**reqwest/tokio 严格隔离在 adapter 与 server、绝不泄漏回核心 wasm**(端上本地 LLM 标注 future,见 #20)。无任何观察与"Rust 作可移植核心语言"的建议相悖。

剩余为**广度**(iOS 真机签名分发、完整 Component Model `cargo-component` guest 固化)与**量化指标**,非根本性未知。详见 [05-roadmap](05-roadmap.md)。
