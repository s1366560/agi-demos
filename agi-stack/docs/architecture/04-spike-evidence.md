## 4. 总评

make-or-break 风险(**运行时无关核心 → 一份代码,server + 浏览器 + 设备**)已**确认**,有跨 server / 浏览器-WASM / 嵌入式 DB / 原生移动绑定的可运行、可测试产物,**移动两端(Android `.so` + iOS XCFramework)均已交叉编译为真实设备产物,iOS 更在 iPhone 17 模拟器内实跑通过**(见 #12/#13)。**PC 桌面外壳**(Tauri)亦跑通:同一核心 + SQLite 设备适配器链入桌面外壳、`ingest`/`search` 命令经 `invoke` 调用、命令逻辑无头可测(见 #17)。**可扩展性轴**亦确认:不可信工具经 `ToolHost` 端口沙箱化,其 WASM 宿主编到每个目标(含浏览器核心)。**热插拔轴**进一步确认:工具表 `ArcSwap` 原子换 + 扩展清单 enable/disable + 形态分类均跑通,且飞行中轮次按轮次边界见旧版本(确定性强于 OpenClaw 的重启式模型,见 [07](07-plugin-runtime-architecture.md))。**控制流/数据流分离轴**亦确认:控制面 SSOT + 数据面声明式 level-triggered reconcile + xDS 风格 version/nonce + ACK/NACK + last-good 跑通,reconciler 纯同步(无 tokio)故端上同实现;此即 Istio 的 Rust 数据面 ztunnel 同构机制(见 [08](08-control-data-plane-separation.md))。**不可信工具运行时**进一步生产化:server/桌面换入 **Wasmtime** 宿主,以 **fuel + epoch 双配额**硬约束不可信 wasm guest,契约升格为 typed `wit/tool.wit`,iOS/浏览器留 Wasmi 兜底(见 #18)。**端上向量检索**亦退役玩具实现:设备索引换入纯 Rust **HNSW 近似最近邻**(`instant-distance`,非 `sqlite-vec` C 扩展以保跨端可移植),embedding 升为高维字符 n-gram 哈希 —— N=10k 实测 P50 **2.43 ms**(暴力 75 ms,31× 加速),稳达 ≤20 ms 阈值(见 #19)。**AI/LLM 抽象轴**亦从 stub 落到真实端点:`LlmPort`/`EmbeddingPort` 新增打 OpenAI/LiteLLM 兼容端点的 HTTP 适配器,cloud↔local 经 DI 环境变量切换(缺省离线 stub、零 key),mock-server 测试全绿,**reqwest/tokio 严格隔离在 adapter 与 server、绝不泄漏回核心 wasm**(端上本地 LLM 标注 future,见 #20)。**local-first 同步轴**亦确认:数据面 payload 经纯核心 `reconcile`(version-vector delta + LWW-register 总序)做离线分叉→重连→最终一致,merge 满足 CRDT 交换/结合/幂等律,core-only 零 tokio/std::time、同编 `wasm32`(见 #21)。**可扩展性轴收尾**:L2 **Skill 引擎**落地 —— Skill = 纯数据(serde,ship 端上为配置)+ **沙箱化 Rhai 触发器**(指令预算 50k、禁 `eval`、无墙钟/std::time),fired 后组合注册中心已准入的 L1 工具;信任轴守恒(配置层无原生码、只能编排已准入工具),语义路由仍归 agent(Agent First),Rhai 经 `getrandom` `wasm_js` 后端同编浏览器 wasm,9 测试绿(见 #23)。**最终,go/no-go 评分卡(#22)收口**:一个可运行 bench 实测核心延迟 + 实读产物体积,联合前 21 项已验证产物组成 14 行风险表,**14/14 ✅、自动判定并打印「建议:GO」、退出码 0** —— 全部 make-or-break 风险已用可运行、可测试的产物退役。无任何观察与"Rust 作可移植核心语言"的建议相悖。

剩余为**广度**(iOS 真机签名分发、完整 Component Model `cargo-component` guest 固化、Python 后端逐能力绞杀替换)而非根本性未知,已在评分卡中显式标注 future。详见 [05-roadmap](05-roadmap.md)。
